판매 설정을 보기 전에 — 설정하는 '위치'부터 달라졌습니다. 흩어져 있던 클래스 관리·설정이 사이트 관리 한 곳으로 모였습니다.

목록과 상세 설정이 같은 공간에 — 페이지 이동 없이 한 흐름으로 이어집니다.
→그 한 공간 안에서, 판매 유형 · 가격 · 할인 · 기간 · 수량을 '판매 설정' 한 화면에서 구성합니다.

새로워진 라이브클래스 · 판매 설정

한 상품에서, 더 유연해진 판매

상품을 복제하지 않고도 여러 판매 버전을, 부가 상품까지 — 한 페이지 안에서 운영합니다.

판매 조건 신규

상품 하나로
여러 판매 버전을

예전엔 없던 개념입니다. 상품을 복제하지 않고도, 한 상품 페이지에서 얼리버드·정가·기수별 조건을 여러 개 동시에 운영합니다.

추가 구성 확장

부가 상품을
함께 판매

본 상품 외 코칭권·파일 등을 함께 제안해 객단가를 높입니다.

판매 조건 카드 하나가 판매 전략 하나입니다.
카드 아래 [+ 판매 조건]을 누르면 카드가 계속 추가되어 — 상품 하나로 여러 전략을 동시에 운영합니다.
→구매자는 한 페이지에서 조건을 비교하고 선택합니다. 이탈 없이 결제로 이어집니다.
바뀐 표현

강의에만 맞던 표현을, 상품 보편적인 표현으로 정비했습니다

라이브클래스에서 다루는 상품의 형태는 점점 다양해지고 있습니다. 그래서 강의에만 들어맞던 표현들을, 어떤 상품에도 두루 쓸 수 있는 표현으로 바꿔가고 있습니다.

클래스 → 상품 모집정원 → 판매 수량 제한
클래스 → 상품은 메뉴(클래스 관리 → 상품 관리)·생성 버튼(클래스 생성 → 상품 생성)·설정 버튼([클래스 설정] → [상품 설정])·상세 페이지 명칭 전반에 걸쳐 바뀝니다. 강의 외 실물·챌린지 등 다양한 판매 대상을 포괄하기 위한 정비로, 기능·동작은 그대로입니다.

구매자 화면 · 상품 상세 페이지

설정한 판매 조건 · 추가 구성은, 구매자에게 이렇게 보입니다

관리자가 설정한 판매 조건과 추가 구성이, 상품 상세 페이지에서 구매자에게 어떻게 보이는지 확인합니다.

① 상세 페이지 — 판매 조건
1설정한 여러 판매 조건(30·60·90일 수강권 등)이 카드로 나열됩니다. 구매자는 원하는 조건을 골라 바로 신청합니다.
② 구매 옵션 선택 — 판매 조건 + 추가 구성
2선택 창에서 판매 조건(왼쪽 카드)과 추가 구성(포트폴리오 리뷰·추가 구성 상품·PDF 모음 등)을 함께 담아 결제합니다.
→관리자 화면에서 만든 '판매 조건'과 '추가 구성'이 구매자 화면에 그대로 이어집니다. 계속해서 각 항목이 어떤 1.0 설정에서 왔는지 살펴봅니다.
