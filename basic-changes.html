기존 고객 안내 · 기본 서비스 변경점

새로워진 라이브클래스, 이렇게 달라집니다

새로 추가된 '판매 설정'은 별도 안내로 다룹니다. 이 문서는 기존에 쓰시던 기능이 새로워진 라이브클래스에서 어떻게 바뀌는지 — 이름·위치·상태 체계 중심으로 정리했습니다. 기능과 데이터는 그대로 유지됩니다.

1

메뉴·용어 개편 — '클래스'가 '상품'으로

관리 메뉴·생성/설정 버튼의 명칭이 상품 중심으로 바뀌고, 상품 유형이 4종으로 세분화됩니다.

2

상품 상태 체계 신설

설정 중 · 판매 중 · 판매 중단 3단계로 상태를 명확히 나눴습니다.

3

노출 설정 · 상품 목록 정비

공개/비공개 용어가 바뀌고, 목록에 '상품 상태' 필터가 생겼습니다.

4

결제 페이지 URL 변경 확인 필요

결제 페이지 주소 구조가 바뀝니다. GA·Meta 등 트래킹 설정을 쓰신다면 점검이 필요합니다.

메뉴 · 용어 개편

'클래스'가 '상품'이 됩니다

강의 외에도 실물·챌린지 등 다양한 판매 대상을 담기 위해, '클래스'를 더 넓은 '상품' 개념으로 확장했습니다. 기능·동작은 그대로입니다.

Before · 클래스 관리
관리 메뉴 = 클래스 관리
After · 상품 관리
관리 메뉴 = 상품 관리 (전체 상품 + 유형별)
바뀐 명칭
클래스 관리→상품 관리 클래스 생성→상품 생성 [클래스 설정]→[상품 설정]
상품 유형이 4종으로 세분화 신규
강의라이브디지털 상품패키지
기존 '프로그램 페이지'에서만 만들던 패키지도, 이제 '사이트 관리 > 상품 관리'에서 함께 생성·관리할 수 있습니다. → 모든 상품을 한 곳에서.

신규 · 상품 상태 체계

상품 상태가 3단계로 명확해집니다

기존 라이브클래스에서 혼용되던 '비공개' 상태를 '설정 중 (미완결 상태)'와 '일부 공개 (비노출)'로 분리해, 상태의 의미를 분명히 했습니다.

설정 중

아직 준비 중인 초안

판매 준비가 완료되지 않은 상태. 상세 페이지가 형성되지 않아 구매자가 접근할 수 없습니다.

판매 중

구매 가능

구매자가 상품·패키지를 구매할 수 있는 상태입니다.

판매 중단

신규 판매만 중단

크리에이터가 신규 판매를 멈춘 상태. 기존 수강생은 영향 없이 이용하며, 신규 결제만 막힙니다.

시점BeforeAfter
최초 생성'비공개'로 생성 — 사이트에 안 보이지만 URL로 접근·구매 가능설정 중 으로 생성 — 상세 페이지 미형성, 구매자 접근 불가
판매 시작'비공개' → '공개' 전환 시 사이트 노출'설정 중' → 판매 중 (전체 공개 / 일부 공개 선택)
판매 중단해당 개념 없음'판매 중' → 판매 중단 — 기존 수강생 유지, 신규 결제만 차단

노출 설정 · 상태 × 노출 조합

'공개/비공개'는 '전체 공개 / 일부 공개'로

노출 여부를 정하는 용어가 더 직관적으로 바뀌었습니다. 동작 방식은 기존 라이브클래스와 같습니다.

공개→전체 공개 비공개→일부 공개 전체 공개 = 사이트에 노출 · 일부 공개 = 사이트에 미노출(URL로만 접근)
상품 상태노출 설정 사이트 노출상세 페이지 진입구매
설정 중—✕✕✕
판매 중전체 공개○○○
판매 중일부 공개✕○○
판매 중단전체 공개○○✕
판매 중단일부 공개✕○✕
※ 노출 설정은 '판매 중'·'판매 중단' 상태에만 적용됩니다. '설정 중' 상태에는 적용되지 않습니다.

상품 목록 화면 변화

목록에서 상태로 관리합니다

비공개 탭 대신 '상품 상태' 필터로 걸러 보고, 유형·공개 상태·판매 상태를 한 화면에서 확인합니다.

1기존 <비공개> 탭→'상품 상태' 필터설정중·판매중·판매중단으로 필터
2목록의 공개 여부 On/Off 토글 제외→[고급 설정]에서 공개 상태 변경
3목록 컬럼 정리 — 가격 · 수강생 수/모집 정원 · 모집 기간 컬럼 제외
※ 한 상품에 여러 판매 조건(서로 다른 가격·수량·기간)이 적용될 수 있어, 하나로 특정되는 값은 목록에서 뺐습니다.

결제 페이지 · 개발 / 트래킹 영향

결제 페이지 URL 구조가 바뀝니다

'판매 조건' 개념이 도입되면서 결제 페이지 구조가 새로 설계되어, 주소가 달라집니다.

!결제 페이지 URL 변경
/classes/{클래스id}/payment?id=… → /order/{주문id}?mkt_product_seq={클래스id}
•트래킹 설정 점검: GA·Meta 등에서 기존 1.0 결제 페이지 URL로 커스텀 이벤트(전환 등)를 설정했다면, 신규 URL 구조로 수정해야 합니다.
•결제 다이렉트(딥링크) URL 제공 불가: 판매 조건 없이는 결제 페이지에 진입할 수 없고, URL에 주문 ID가 없으면 상품 정보를 조회할 수 없어 — 기존처럼 상품 결제 페이지로 바로 보내는 고정 URL은 제공되지 않습니다.
※ 이 변경은 트래킹·딥링크를 직접 설정해 쓰시는 경우에만 영향이 있습니다. 일반적인 판매·구매 흐름에는 영향이 없습니다.
