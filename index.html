신규 · 상품 상태 체계

상품 상태가 3단계로 명확해집니다

기존 라이브클래스에서 혼용되던 '비공개' 상태를 '설정 중 (미완결 상태)'와 '일부 공개 (비노출)'로 분리해, 상태의 의미를 분명히 했습니다.

설정 중

아직 준비 중인 초안

판매 준비가 완료되지 않은 상태. 상세 페이지가 형성되지 않아 구매자가 접근할 수 없습니다.

판매 중

구매 가능

구매자가 상품·패키지를 구매할 수 있는 상태입니다.

판매 중단

신규 판매만 중단

크리에이터가 신규 판매를 멈춘 상태. 기존 수강생은 영향 없이 이용하며, 신규 결제만 막힙니다.

구매자 화면 · 상품 상세 페이지

설정한 판매 조건 · 추가 구성은, 구매자에게 이렇게 보입니다

관리자가 설정한 판매 조건과 추가 구성이, 상품 상세 페이지에서 구매자에게 어떻게 보이는지 확인합니다.

① 상세 페이지 — 판매 조건
1설정한 여러 판매 조건(30·60·90일 수강권 등)이 카드로 나열됩니다. 구매자는 원하는 조건을 골라 바로 신청합니다.
② 구매 옵션 선택 — 판매 조건 + 추가 구성
2선택 창에서 판매 조건(왼쪽 카드)과 추가 구성(포트폴리오 리뷰·추가 구성 상품·PDF 모음 등)을 함께 담아 결제합니다.
→관리자 화면에서 만든 '판매 조건'과 '추가 구성'이 구매자 화면에 그대로 이어집니다. 계속해서 각 항목이 어떤 1.0 설정에서 왔는지 살펴봅니다.

결제 페이지 · 개발 / 트래킹 영향

결제 페이지 URL 구조가 바뀝니다

'판매 조건' 개념이 도입되면서 결제 페이지 구조가 새로 설계되어, 주소가 달라집니다.

!결제 페이지 URL 변경
/classes/{클래스id}/payment?id=… → /order/{주문id}?mkt_product_seq={클래스id}
•트래킹 설정 점검: GA·Meta 등에서 기존 1.0 결제 페이지 URL로 커스텀 이벤트(전환 등)를 설정했다면, 신규 URL 구조로 수정해야 합니다.
•결제 다이렉트(딥링크) URL 제공 불가: 판매 조건 없이는 결제 페이지에 진입할 수 없고, URL에 주문 ID가 없으면 상품 정보를 조회할 수 없어 — 기존처럼 상품 결제 페이지로 바로 보내는 고정 URL은 제공되지 않습니다.
※ 이 변경은 트래킹·딥링크를 직접 설정해 쓰시는 경우에만 영향이 있습니다. 일반적인 판매·구매 흐름에는 영향이 없습니다.
